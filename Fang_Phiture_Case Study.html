# Rate card
base_fare = 2
cost_per_mile = 1.15
cost_per_min = 0.22
min_fare = 5
max_fare = 400
service_fee = 1.75 
kiwi_revenue_percentage = 0.20
gross_margin = 0.5  # Example value, adjust based on actual data


# Fare calculation formula
## 1. $2.00 + Duration(min) * 0.22 + Distance(miles) * 1.15 
## 2. Add prime-time if applicable 
## 3. Enforce $5 minimum and $400 maximum 
## 4. Add 1.75 Service Fee 

# Add a new column to calculate the fare
# 1. Initial fare calculation
ride_ids_df['fare'] = base_fare + (ride_ids_df['ride_duration_minutes'] * cost_per_min) + (ride_ids_df['ride_distance_miles'] * cost_per_mile)
# 2. Prime time calculation: fare = fare + (fare * (ride_prime_time / 100)) -> fare += (fare * (ride_prime_time / 100))
ride_ids_df['fare'] += ride_ids_df['fare'] * (ride_ids_df['ride_prime_time'] / 100)
# 3. Check the minimum and maximum limit, then adds a service fee
# 3.1 Use lambda function to make sure the fare is within the limit
# 3.1-1 min(x, max_fare): Catch the fare at the maximum fare limit if x exceeds it.
# 3.1-1 max(min_fare, min(x, max_fare)): Catch the fare at the minimum fare limit if x belows it.
ride_ids_df['fare'] = ride_ids_df['fare'].apply(lambda x: max(min_fare, min(x, max_fare))) + service_fee


# Revenue calculation formula: value for Kiwi is 20% of the fare, excluding the service fee
# Add a new column to calculate the kiwi_revenue
ride_ids_df['kiwi_revenue'] = (ride_ids_df['fare'] - service_fee) * kiwi_revenue_percentage


# Calculate total revenue & totla_drivers & ARPU
total_revenue = ride_ids_df['kiwi_revenue'].sum()
total_drivers = driver_ids_df['driver_id'].nunique()
ARPU = total_revenue / total_drivers
print("ARPU: $", ARPU.round())


# Calculate churn rate : define churned drivers as those who haven't had any rides in the last 30 days
# Merge ride_timestamps & ride_id to get the driver_id
ride_driver_timestamps = ride_ids_df.merge(ride_timestamps_df, how='left', on='ride_id')
# Assume the latest date in timestamp as deadline and calculate prior 30 days as a baseline for a churn date
churn_date = ride_driver_timestamps['timestamp'].max() - pd.Timedelta(days=30)
# Calculate the last ride dates for each driver and compare it to churn date to see if the driver churned or not
last_ride_dates = ride_driver_timestamps.groupby('driver_id')['timestamp'].max().reset_index()
last_ride_dates.rename(columns={'timestamp': 'last_ride_date'}, inplace=True) #rename the column to be more specific
# Define churned drivers and count # of churned frivers
churned_drivers = last_ride_dates[last_ride_dates['last_ride_date'] < churn_date]
churned_drivers_count = churned_drivers['driver_id'].nunique()
# Churn rate = total of lost users / total users 
churn_rate = churned_drivers_count / total_drivers
print("Churn rate: ", round(churn_rate * 100,2),'%')

LTV = (ARPU * gross_margin) / churn_rate
print("Average Lifetime Value (LTV): $", LTV.round())



# Visualize the driver lifetimes values distribution -> create a dataframe
# Lifetime calculation -> Merge last_ride_dates with driver_ids table to get onboarddate from each driver
driver_lifetime = driver_ids_df.merge(last_ride_dates, how='left', on='driver_id')

# Calculate the lifetime of each driver in days
driver_lifetime['lifetime_days'] = (driver_lifetime['last_ride_date'] - driver_lifetime['driver_onboard_date']).dt.days


# Calculate ARPU for each driver
total_revenue_per_driver = ride_ids_df.groupby('driver_id')['kiwi_revenue'].sum().reset_index()
# Merge total_revenue_per_driver & driver_lifetime into one table
driver_ltv_df = driver_lifetime.merge(total_revenue_per_driver, how='left' , on='driver_id')

# Calculate LTV for each driver -> LTV = (avg.revenue) * lifetime

1-1. Q:Any main factors affect a driver’s lifetime value?

  • Why I want to know?
    • Understanding what drives higher LTV
  • So what?
    • tailor strategies to increase driver satisfaction and longevity, leading to higher profits.

Findings:

  • Ride Count:

    • Coefficient: -0.4375
    • Significance: Highly significant (p-value < 0.001)
    • Impact: Negative
    • Interpretation:
      • The number of rides completed by a driver negatively impacts the LTV.
      • For each additional ride, the LTV decreases by approximately 0.4375 units.
      • This counterintuitive result might suggest diminishing returns for additional rides or potential burnout effects on drivers.

  • Total Earnings:

    • Coefficient: 0.2500
    • Significance: Highly significant (p-value < 0.001)
    • Impact: Positive
    • Interpretation:
      • Total earnings positively impact the LTV.
      • For each unit increase in total earnings, the LTV increases by approximately 0.25 units.
      • This indicates that higher earnings are associated with higher LTV, reflecting the financial motivation for drivers to continue working.

  • Average Wait Time Before Arrived:

    • Coefficient: -8.891e-14
    • Significance: Significant (p-value < 0.001)
    • Impact: Negative, but very small
    • Interpretation:
    • The average wait time before arrived has a very small negative impact on LTV.
    • This coefficient is almost negligible, indicating that wait time before arrived does not significantly affect LTV in a practical sense.

Recommendations:

  • Optimize Ride Assignments:

    • Focus on optimizing ride assignments and balance the number of rides and provide adequate breaks to maintain driver satisfaction and performance.

  • Increase Earnings Opportunities:

    • Create incentive programs that reward drivers for reaching certain milestones, such as a high number of completed rides or positive customer feedback.
    • Peak Time Bonuses: Offer additional earnings opportunities during peak hours or high-demand periods.

  • Reduce Wait Times:

    • Continue efforts to minimize wait times through efficient dispatch systems and better matching of drivers to passengers. Reducing idle time can improve the overall driver experience and contribute to retention.
